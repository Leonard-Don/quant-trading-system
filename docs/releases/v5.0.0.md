# v5.0.0

发布日期：2026-04-18

## 升级摘要

- 公开仓正式收敛为 `策略回测`、`实时行情`、`行业热度` 三块 GitHub-facing 能力。
- `定价研究`、`上帝视角`、`研究工作台` 与 `Quant Lab` 已迁移到私有 companion repo `super-pricing-system`。
- 公开前端入口、公开后端路由、README、API 文档与测试元数据均已按新的双仓边界同步收口。

## 重点变更

### 1. 公开仓边界调整

- 前端公开 view 只保留 `backtest`、`realtime`、`industry`。
- 访问历史系统页链接时，会自动回落到 `backtest`，避免旧 deep link 直接失效。
- `cross-market` 继续保留在公开仓，作为回测能力的一部分维护。

### 2. 系统能力迁移

- 以下能力不再由公开仓提供：`定价研究`、`上帝视角`、`研究工作台`、`Quant Lab`。
- 以下公开路由已不再挂载：`/pricing/*`、`/macro*`、`/research-workbench/*`、`/quant-lab/*`、`/alt-data/*`。
- 系统侧继续由私有仓 `super-pricing-system` 承接，并以 GitHub private repo 形式维护。

### 3. 兼容与文档

- `POST /realtime/alerts/hits` 仍接受 `create_workbench_task` 字段以兼容旧客户端，但公开仓会忽略该值，不再创建系统侧研究任务。
- OpenAPI、API Reference 与 Postman collection 已同步更新兼容字段说明。
- E2E package 名称已从旧的 `pythonproject` 收口到当前双仓命名。

## 升级提示

- 如果你只使用公开仓，请将导航与集成入口调整为 `backtest / realtime / industry` 三块能力。
- 如果你仍需要系统模块，请切换到私有仓 `super-pricing-system` 继续开发，不要再依赖公开仓中的旧系统页或旧系统路由。
- 升级后建议重新验证实时提醒、行业热度和跨市场回测的核心工作流，确认旧兼容入口已符合预期。
